# AI Technology Radar with RAG and Lightweight Agents

## 项目简介

本项目是一个面向 AI Lab / 技术团队的 AI 技术情报分析系统原型，用于自动采集、整理和评估最新 AI 技术动态，并辅助判断某项技术是否值得进一步跟进或投入 PoC 验证。

系统围绕“信息采集 → 技术画像 → 多维评分 → RAG 检索 → Agent 分析 → 报告生成”构建完整流程，尝试解决 AI 技术趋势跟踪中信息分散、判断主观、复盘困难和优先级不清晰的问题。

本项目不是简单的论文摘要工具，而是将结构化证据、规则评分、RAG 检索和轻量 Agent 工作流结合起来，形成一个可解释、可复核、可扩展的技术情报分析原型。

---

## 项目背景

在 AI 技术快速发展的背景下，团队每天都会面对大量新论文、新模型、新框架和开源项目。仅依靠人工浏览 arXiv、GitHub、Hugging Face 或社区资讯，容易出现以下问题：

- 信息来源分散，难以及时发现高价值技术；
- 判断标准不统一，容易受标题、热度或主观印象影响；
- 缺少历史经验沉淀，类似技术可能被重复评估；
- 技术是否值得投入 PoC 缺少清晰依据；
- 技术周报和技术雷达整理成本较高。

因此，本项目设计了一个 AI 技术情报雷达系统，用于辅助完成技术初筛和决策支持。

## 核心功能

### 1. 多源信息采集

系统支持从多个公开来源采集 AI 技术相关信息，包括：

- arXiv 论文信息；
- GitHub 开源项目信息；
- Hugging Face 模型信息；
- 社区讨论与技术动态。

采集后，系统会将不同来源的信息整理为统一格式，方便后续分析。

---

### 2. 技术画像构建

系统会为每个技术条目构建结构化画像，包括：

- 技术名称；
- 来源平台；
- 技术摘要；
- 发布时间；
- GitHub Star；
- 是否开源；
- 是否有论文；
- 是否有模型或 Demo；
- 是否包含 Benchmark；
- 是否存在 License 或部署风险。

这些信息会作为后续评分和分析的基础。

---

### 3. 多维评分机制

本项目采用可解释的加权评分机制，而不是直接让大模型判断“是否值得跟进”。

评分维度包括：

| 评分维度 | 权重 | 说明 |
| --- | ---: | --- |
| 相关性 | 20% | 是否匹配当前关注方向 |
| 工程可行性 | 25% | 是否有代码、模型、API 或部署线索 |
| 可复现性 | 20% | 是否有 Benchmark、数据集或实验细节 |
| 社区 / 学术热度 | 15% | GitHub Star、下载量、引用量、社区讨论 |
| 新颖性 | 10% | 是否代表新方法、新框架或新路线 |
| 风险控制 | 10% | License、隐私、合规和部署成本风险 |

最终系统会根据综合分数给出行动建议：

| 分数区间 | 建议 |
| --- | --- |
| 80 分以上 | 立即跟进，建议投入 PoC |
| 65 - 80 分 | 持续观察，补充证据后再判断 |
| 50 - 65 分 | 低优先级归档 |
| 50 分以下 | 暂不投入 |

---

### 4. RAG 检索增强分析

系统会维护一个历史知识库，用于存储过往分析结果和类似技术案例。

当评估一项新技术时，系统可以通过 RAG 检索历史相似技术、过往 PoC 结论和失败原因，从而辅助生成更有上下文的判断。

在本项目中，RAG 不直接替代决策，而是用于增强解释和辅助复核。

---

### 5. 轻量 Agent 工作流

项目中设计了多个轻量 Agent，用于完成边界清晰的专项任务：

- Paper Reading Agent：提取论文贡献点、实验设置和局限；
- Repo Inspection Agent：检查代码仓库、License、安装方式和维护情况；
- PoC Planning Agent：生成小规模验证计划；
- Report Writer Agent：生成技术分析报告。

Agent 的输出需要绑定原始证据或结构化字段，避免完全依赖模型自由发挥。

## 项目亮点

### 1. 从信息收集走向决策辅助

本项目不是简单聚合论文或开源项目，而是进一步抽取结构化证据，并通过多维评分辅助判断技术投入优先级。

### 2. 评分过程可解释

系统没有直接让大模型输出最终结论，而是通过明确的评分维度和权重计算结果，使每个推荐结论都可以被追溯和复核。

### 3. 结合 RAG 与历史经验

通过历史知识库和相似案例检索，系统可以把新技术放到已有技术路线和过往经验中进行比较，避免孤立判断。

### 4. Agent 只做专项任务

项目没有盲目使用复杂多 Agent 架构，而是采用“规则主流程 + 专项 Agent 辅助”的设计，使系统更容易落地和调试。

### 5. 具备扩展空间

当前版本是一个 MVP 原型，后续可以扩展为完整的 AI 技术雷达平台。

- 增加人工反馈机制，支持修正评分和优化规则；
- 增强 GitHub 仓库质量分析，例如 issue 活跃度、commit 频率；
- 增加 License、隐私和合规风险识别；
- 支持自动生成技术周报。
